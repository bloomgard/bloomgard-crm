# Bloomgard ERP Context

## Project Overview
Bloomgard is a modular ERP system for enterprise quotation management, built using a "Split Brain" architecture.
- **Web App:** Next.js (App Router), deployed on Vercel. Handles API routes, PDF generation, and database interactions.
- **Mobile App:** React Native with Capacitor, designed to be deployed as an Android APK.

## Architecture & Structure
- `/web`: Root directory for the web app. Contains `app/`, `components/`, and `utils/`.
- `/mobile`: Root directory for the mobile app. Uses Capacitor plugins for native device capabilities.
- **Database:** Supabase (PostgreSQL). We use Service Role keys for backend API routes and Anon keys for frontend.
- **AI Integration:** Uses OpenRouter API for LLM tasks (data analysis, insight generation, and auto-follow-up).

## Critical Rules for the AI
1. **Separation of Concerns:** - Never add Capacitor imports (like `@capacitor/core`) to files inside `/web`.
   - The web app must use standard web APIs (fetch, window.print, etc.).
   - Mobile app files can use Capacitor plugins.
2. **Database:** Always use Supabase `tenant_id` to filter queries. 
3. **UI Style:** Currently transitioning to a Glassmorphism UI using Tailwind CSS (`backdrop-blur`, `bg-white/10`, etc.).
4. **Follow-Up Feature:** We are implementing an autonomous Agent System:
   - Alert System: Triage pending quotes based on `follow_up_due_date`.
   - Agent System: Triggered by `/api/trigger-agent`. Needs to pull quote context and use LLMs for automated email drafts.

## Current Technical Stack
- Framework: Next.js 16
- Styling: Tailwind CSS 4
- Database: Supabase
- Auth: Supabase Auth
- AI Model: OpenRouter (GPT-3.5/4/Claude)
- Email: Resend / Nodemailer
